## 阶段 1：数据源差异与标准口径

### 本阶段目标

- 明确当前项目与 `TradingAgents-CN` 的数据分析来源是否一致
- 明确哪些层是一致的，哪些层存在明显差异
- 固化当前项目在 A 股主链下的“标准口径”，避免后续在数据源上摇摆

### 执行结论

结论：`不完全一致`

如果只看当前主要使用的 `A 股单股分析主链`，两边有一部分重合；  
如果看完整的数据治理和分析来源体系，差异仍然比较大。

### 当前标准口径

当前项目后续默认采用这套 A 股主链标准口径：

- 行情 / 股票池：
  - `AKShare + CSV 缓存`
- 基本面 / 估值：
  - `AKShare + SQLite 快照`
- 事件驱动：
  - `公告 + 业绩预告 + 财报预约披露`
- 市场情绪：
  - `全市场宽度情绪`
- 行业归属 / 行业横向：
  - `AKShare / cninfo 行业路径 + SQLite 同行快照`

### 差异清单

| 分析层 | 当前项目 | TradingAgents-CN | 是否一致 | 当前处理建议 |
| --- | --- | --- | --- | --- |
| A 股行情 / 股票池 | `AKShare` 为主，带 CSV 缓存 | `AKShare / Tushare / BaoStock / MongoDB` 多级治理 | `部分一致` | 保持当前主链不动，先不强行改成多源 |
| 基本面 / 估值 | `AKShare` 为主，SQLite 快照 | `MongoDB -> Tushare / AKShare` 降级 | `部分一致` | 先保留 AKShare 主链，后续补 Tushare 备源 |
| 事件驱动 | 公告、业绩预告、披露预约 | 更偏“新闻 + 事件 + 数据源切换” | `不一致` | 当前先保留事件面定义 |
| 新闻层 | 当前没有独立实时新闻主链 | Finnhub / Alpha Vantage / NewsAPI / 中文财经新闻 | `不一致` | 这是后续最值得补的一层 |
| 市场情绪 | 全市场涨跌家数、涨停跌停宽度 | 市场 + 新闻 + 社交情绪更混合 | `不一致` | 当前继续用市场宽度情绪作为主口径 |
| 行业归属 / 横向比较 | AKShare / cninfo + 同行快照 | 多源 provider + 更强数据治理 | `部分一致` | 先保持当前插件链，不急着改多源 |

### 本阶段执行结果

- 已确认：
  - 两边在 `A 股核心事实数据` 上并非完全冲突
  - 但在“数据源治理方式、新闻层、情绪层”上口径明显不同
- 已决定：
  - 当前项目短期不追求和 `TradingAgents-CN` 完全同口径
  - 先固定当前 A 股主链标准口径，再按价值补关键数据层

### 本阶段状态

- 当前状态：`已执行`
- 执行日期：`2026-04-19`

## 阶段 2：补关键数据层

### 本阶段目标

在不打乱当前主链的前提下，只补最值得学的增量数据层。

### 执行顺序

1. `A 股实时新闻层`
   - 目标：
     - 补足当前只有“事件面”、缺少“实时新闻流”的短板
   - 预期收益：
     - 让事件驱动和新闻驱动分开表达
     - 提升催化识别和短期扰动识别能力

2. `Tushare 基本面备源`
   - 目标：
     - 不替换当前 AKShare 主链
     - 只作为基本面和估值备份 / 校验源
   - 预期收益：
     - 减少单一源风险
     - 提高基本面快照的稳健性

3. `行业景气 / 板块热度补充`
   - 目标：
     - 让行业横向比较不只看同行财务快照
   - 预期收益：
     - 提升行业比较层的景气解释力

### 本阶段状态

- 当前状态：`已执行`
- 已完成：
  - `A 股实时新闻层` 第一版已接入
  - 当前使用 `akshare.stock_news_em + SQLite 新闻快照`
  - 单股页已新增“实时新闻摘要”，与事件驱动层分开表达
  - `Tushare 基本面备源` 第一版已接入
  - 当前采用 `AKShare 主源 + Tushare 备源 + SQLite 快照`
  - `行业景气 / 板块热度补充` 已接入，并作为行业横向比较的第 4 个子模块
- 仍未完成：
  - 暂无
- 下一步执行建议：转入阶段 3，继续把执行层下沉到候选池

## 阶段 3：执行层下沉到候选池

### 本阶段目标

  - `execution_risk_score`

  - `target_price_range`
- 候选排序升级为：
  - 研究质量
  - 可执行性

### 本阶段状态

- 当前状态：`已执行`
- 已完成：
  - 量价代理候选与中线候选已新增：
    - `执行置信度`
    - `执行风险分`
    - `执行得分`
    - `风险修正动作`
    - `建议目标仓位`
    - `目标价区间`
  - 候选排序开始把“研究分 + 执行得分 + 风险修正”一起纳入

## 阶段 4：执行层下沉到组合与回测

### 本阶段目标

让组合层和回测层真正使用交易判定层字段，而不只停留在单股页展示。

### 目标能力

- 组合层：
  - 用执行风险分辅助排序
  - 用目标价情景辅助仓位建议
  - 用风险修正动作辅助调仓建议
- 回测层：
  - 记录执行计划
  - 记录风险修正动作
  - 支持事后回答“当时为什么买 / 卖 / 降级”

### 本阶段状态

- 当前状态：`已执行`
- 已完成：
  - 组合约束新增：
    - `最低执行置信度`
    - `最高执行风险分`
  - 组合权重开始参考：
    - 综合得分
    - 执行置信度
    - 执行风险分
  - 当前推荐与回测调仓记录开始保留：
    - `risk_adjusted_action`
    - `position_guidance`
    - `target_price_range`
    - `execution_confidence`
    - `execution_risk_score`
- 下一步执行建议：后续可转入因子归因、状态变化跟踪或更完整组合风控
