# 股票分析系统专业化修补追踪表

这份文档用于持续记录系统从“多维选股工具”向“更专业的投研分析系统”演进的状态。

文档入口：
- 首次阅读或准备更新 `docs/` 时，请先看 [README.md](../README.md)。
- 固定更新模板请看 [DOC_UPDATE_TEMPLATES.md](../templates/DOC_UPDATE_TEMPLATES.md)。
- 字段含义不清楚时，请先查 [FIELD_GLOSSARY.md](./FIELD_GLOSSARY.md)。

规则：
- 每次增加或修改会影响研究结论、候选筛选、组合选股、风险判断、事件解释、市场环境判断的功能时，必须同步更新本文件。
- 每次更新至少要写清楚：
  - 本次新增了什么
  - 本次没有覆盖什么
  - 哪些模块已经统一
  - 哪些模块仍然缺失
- 如果本次改动影响股票最终判断逻辑，还必须同步更新 [STOCK_ANALYSIS_LOGIC.md](./STOCK_ANALYSIS_LOGIC.md)。
- 如果本次改动影响模块职责、数据流、插件结构、UI入口或存储结构，还必须同步更新 [CODE_ARCHITECTURE.md](./CODE_ARCHITECTURE.md)。
- 如果需要从机构使用者视角持续维护“当前缺陷清单”，同步更新 [PRIVATE_FUND_GAP_BACKLOG.md](./PRIVATE_FUND_GAP_BACKLOG.md)。

## 核心输出字段中文说明

- 技术面结论（`technical_recommendation`）：技术层面对个股的直接判断
- 基本面评分（`fundamental_score`）：财务质量、估值与增长的综合分数
- 市场情绪状态（`market_sentiment_state`）：当前市场环境的强弱状态
- 事件驱动评分（`event_score`）：近期事件面对个股的综合影响分
- 行业横向比较评分（`industry_comparison_score`）：个股在行业内的相对位置分
- 最终推荐结论（`recommendation`）：主链汇总后的最终推荐级别
- 最终结论依据（`final_decision_basis`）：最终推荐为什么成立的摘要说明

## 双轴状态说明

本文件现在把每个能力拆成两个维度分别跟踪：

- `系统接入状态`
  - `未开始`：还没有设计或落地
  - `进行中`：已经有部分实现，但还没稳定进入主流程
  - `已接入`：已经进入主流程，但还不够完善
  - `已统一`：已经进入主流程，且与统一评分口径保持一致
- `机构成熟度`
  - `未开始`：离机构使用标准还有明显距离
  - `进行中`：已有雏形，但可信度、深度或工作流仍不足
  - `已接入`：已经能支持部分机构使用场景，但还不够完整
  - `已统一`：不仅已接入，而且基本达到当前系统目标下的机构化标准
